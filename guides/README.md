# guides/

Step-by-step integration and implementation guides.

## What goes here

- Actionable instructions with ordered steps someone can follow
- Integration guides for connecting tools to our stack
- Setup procedures, migration playbooks, configuration walkthroughs
- Must have prerequisites, steps, and a verification method

## What does NOT go here

- Technology explanations without actionable steps → `references/`
- Tool evaluations → `evaluations/`
- Broad surveys → `landscapes/`

## Required sections

Prerequisites, Steps, Verification, Troubleshooting, Key Sources

## Naming

`[action-or-topic-slug].md` — e.g., `railway-deploy-cron.md`, `supabase-rls-setup.md`
